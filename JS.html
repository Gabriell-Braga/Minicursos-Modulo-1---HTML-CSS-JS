MÓDULO 3:
JAVASCRIPT NA PRÁTICA

INTERAÇÃO, DOM E PRIMEIROS PASSOS NO NAVEGADOR

O Papel do JavaScript

Da página estática ao comportamento inteligente.

O TRIPÉ COMPLETO

  • HTML: estrutura e significado.
  • CSS: aparência e layout.
  • JavaScript: lógica, eventos e atualização da página.

O QUE O JS FAZ?

DOM

Ler e alterar elementos da página após o carregamento.

EVENTOS

Responder a cliques, teclas, rolagem e envio de formulários.

DADOS

Calcular, guardar temporariamente no navegador e validar entradas.

ONDE O JS RODA

No navegador, o motor JavaScript (por exemplo V8 no Chrome) executa seu código depois que o HTML foi interpretado. Por isso ligamos o script no final do <body> ou usamos defer na tag <script>.

  • Não precisa instalar nada além do editor e do navegador para os primeiros exercícios.
  • O Console das DevTools mostra erros e o resultado de console.log.

REQUISITOS E FERRAMENTAS

O que você precisa

  • Navegador atualizado (Chrome, Edge, Firefox ou Safari) — o JS roda nele.
  • Editor de código — por exemplo Visual Studio Code ou Cursor (baseado no VS Code).
  • Pasta do projeto aberta no editor; arquivos .html / .js salvos em UTF-8.

Já vem no navegador

  • Ferramentas de desenvolvedor (F12): abas Console, Elements e Sources para inspecionar HTML, CSS e erros de JS.

EXTENSÕES ÚTEIS (VS CODE / CURSOR)

Instale pelo painel de extensões (ícone de quadrados no lado esquerdo). Nomes podem variar levemente; busque pelo título.

LIVE SERVER

Sobe um servidor local e recarrega ao salvar — evita abrir o HTML como file://, que às vezes limita fetch e módulos.

ESLINT

Avisa erros de sintaxe e estilo no próprio editor enquanto você digita.

PRETTIER

Formata JS/HTML/CSS com um atalho — código mais legível e uniforme.

Opcionais: Error Lens (mostra erros na linha), JavaScript (ES6) code snippets (atalhos para laços e funções). No Cursor, muitas extensões do VS Code funcionam igual.

INCLUIR UM ARQUIVO .js

<script src="perfil.js" defer></script>
<!-- defer: baixa em paralelo e executa após o HTML -->

Um arquivo externo mantém HTML, CSS e JS organizados — o mesmo padrão do exemplo Perfil + JS.

CONSOLE E console.log

Abra F12 → aba Console. Use para inspecionar valores e depurar.

console.log('Olá, mundo!');
let n = 2 + 3;
console.log(n);

VARIÁVEIS: let E const

  • let — valor pode mudar (contador, texto atualizado).
  • const — referência fixa; use quando não vai reatribuir.
  • Evite var em código novo — comportamento de escopo mais confuso.
const ano = new Date().getFullYear();

TIPOS ÚTEIS NO INÍCIO

STRING

Texto entre aspas: 'Olá'

NÚMERO

Inteiros e decimais: 2026, 3.14

BOOLEAN

true ou false

REVISÃO DE SINTAXE

Condicionais, laços e tratamento de erro — só um passe rápido.

SINTAXE · IF E ELSE

Mesma ideia de outras linguagens: executa um bloco só se a condição for verdadeira. Encadeie com else if e finalize com else opcional.

if (nota >= 7) {
  console.log('aprovado');
} else if (nota >= 5) {
  console.log('recuperação');
} else {
  console.log('reprovado');
}

// combinar condições
if (logado && idade >= 18) { … }
if (erro || !conectado) { … }

Igualdade estrita: === e !== (compara valor e tipo). Em condições, valores “falsy” incluem 0, '', null, undefined, NaN.

SINTAXE · FOR E WHILE

For com índice é o clássico “contar até N”. For…of percorre valores de um array (ou string). While repete enquanto a condição for verdadeira — cuidado com loop infinito.

for (let i = 0; i < 3; i++) {
  console.log(i); // 0, 1, 2
}

const nomes = ['Ana', 'Bob'];
for (const n of nomes) {
  console.log(n);
}

let n = 0;
while (n < 5) {
  n++;
}

No front-end, laços aparecem ao montar listas no DOM ou ao processar dados — aqui só o desenho da linguagem.

SINTAXE · TRY / CATCH

Código que pode falhar (rede, JSON inválido, sessionStorage bloqueado) vai no try. Se der erro, o fluxo salta para catch. finally roda sempre — útil para fechar recursos.

try {
  sessionStorage.setItem('chave', '1');
  // algo que pode lançar exceção
} catch (err) {
  console.warn('Não foi possível gravar:', err);
} finally {
  // opcional: sempre executa
}

No perfil.js, o try/catch em volta do sessionStorage evita quebrar a página se o navegador bloquear armazenamento.

FUNÇÕES

Agrupam passos reutilizáveis. No perfil.js, a função interna atualizaBotaoTopo roda a cada rolagem:

function atualizaBotaoTopo() {
  var scrollou = window.scrollY > 300;
  btnTopo.hidden = !scrollou;
}

Funções genéricas usam return; esta só atualiza o estado do botão.

DOM: A ÁRVORE DA PÁGINA

O navegador monta uma árvore a partir do HTML. Tudo passa pelo objeto document.

  • Cada tag vira um elemento com propriedades e métodos.
  • Busca por id, classe ou seletor CSS.
// ideia da hierarquia (simplificado)
document
└── html
    ├── head
    └── body
        ├── header#topo
        ├── main … section#sobre
        └── footer#contato

BUSCAR ELEMENTOS

document.getElementById('rodape-ano');
document.querySelector('.cartao');

Prefira getElementById quando há um id único — é direto e rápido.

ALTERAR TEXTO

A propriedade textContent define o texto visível dentro do elemento (seguro para texto simples).

elemento.textContent = 'Novo texto';

CLASSES NO JS: classList

el.classList.add('ativo');
el.classList.remove('ativo');
el.classList.toggle('visivel');

Combine com CSS: o JS só alterna classes; o visual continua no arquivo .css.

EVENTOS: addEventListener

botao.addEventListener('click', function () {
  console.log('Clicou!');
});

Outros eventos comuns: 'scroll', 'keydown', 'submit'.

EVENTO SCROLL

Dispara enquanto a página rola. Lemos window.scrollY e atualizamos o botão (trecho no espírito do perfil.js):

window.addEventListener('scroll', atualizaBotaoTopo, {
  passive: true
});

Use { passive: true } quando não chamar preventDefault — o navegador otimiza a rolagem.

DATA: new Date()

O objeto Date representa um instante. Para o ano atual no rodapé:

new Date().getFullYear();

ARMAZENAMENTO: sessionStorage

sessionStorage guarda pares chave/valor só enquanto a aba está aberta. Fechou a aba, os dados somem — útil para “primeira visita nesta sessão”.

sessionStorage.setItem('chave', 'valor');
sessionStorage.getItem('chave');

localStorage persiste mesmo fechando o navegador — use quando fizer sentido.

ROLAR COM SUAVIDADE

No HTML, links para seções do mesmo arquivo:

<nav aria-label="Ir para seção">
  <a href="#sobre">Sobre</a>
  <a href="#contato">Contato</a>
</nav>

No CSS (global) e no JS (botão):

html { scroll-behavior: smooth; }

alvoTopo.scrollIntoView({
  behavior: 'smooth', block: 'start'
});

BOAS PRÁTICAS RÁPIDAS

  • Script no fim do body ou com defer.
  • IIFE envolve o arquivo para não criar variáveis globais soltas.
  • Se algo falhar, leia o erro no Console.
(function () {
  'use strict';
  // seu código aqui
})();

EXERCÍCIO: PERFIL + JS

Use o mesmo perfil dos módulos anteriores. Referência completa no exemplo exemplos/js/perfil.js. Objetivos:

  • Arquivo perfil.js ligado com defer.
  • Ano no rodapé atualizado com JavaScript.
  • Botão fixo que aparece ao rolar e volta ao topo suavemente.
  • Mensagem diferente na primeira vez que a aba abre a página (sessionStorage).
  • Mini-menu no hero com links para #sobre, #habilidades, #contato.

PASSO 1: HTML · CABEÇALHO E SEÇÕES

Âncoras para o menu e id="topo" para o botão voltar:

<header class="perfil-hero" id="topo">
  <h1>…</h1>
  <nav class="perfil-hero__nav" aria-label="Ir para seção">
    <a href="#sobre">Sobre</a>
    <a href="#habilidades">Habilidades</a>
    <a href="#contato">Contato</a>
  </nav>
</header>

<main class="perfil-main">
  <section class="cartao" id="sobre"> … </section>
  <section class="cartao" id="habilidades"> … </section>
</main>

PASSO 1: HTML · RODAPÉ, BOTÃO E SCRIPT

No rodapé, guardamos os pontos que o JS vai preencher: o parágrafo #msg-visita recebe a mensagem da sessão; o <span id="rodape-ano"> recebe o ano. Fora do footer, o botão #btn-voltar-topo começa com hidden até o script decidir exibi-lo. Por último, o <script defer> carrega perfil.js depois do HTML.

<footer class="perfil-rodape" id="contato">
  … tabela de contato …
  <p class="perfil-rodape__extras" id="msg-visita" role="status"></p>
  <p class="perfil-rodape__copy">
    © <span id="rodape-ano"></span> Nome — …
  </p>
</footer>

<button type="button" id="btn-voltar-topo" class="btn-voltar-topo" hidden
        aria-hidden="true" aria-label="Voltar ao topo">↑</button>

<script src="perfil.js" defer></script>

PASSO 1: CSS · VISÃO GERAL

O arquivo exemplos/js/estilo.css copia o visual do exemplo “Perfil + CSS” e acrescenta regras só para o que o JavaScript e os novos elementos pedem. Assim o HTML/CSS base não mudam à toa: cada bloco novo tem um propósito.

  • Rolagem e menu — scroll-behavior no html e estilos do mini-menu .perfil-hero__nav (links para #sobre, etc.).
  • Mensagem dinâmica — classe .perfil-rodape__extras envolve o parágrafo onde o JS escreve o texto da sessão.
  • Botão “voltar ao topo” — classe .btn-voltar-topo: fixo no canto, circular, com estados de hover e foco.

Os próximos slides detalham cada grupo com o código.

PASSO 1: CSS · ROLAGEM E MENU DO HERO

Com scroll-behavior: smooth no elemento raiz, qualquer navegação por âncora (links do menu ou scrollIntoView no JS) ganha transição suave em vez de “pulo” seco.

O .perfil-hero__nav usa flex para centralizar e envolver os links; cada a vira uma “pílula” com borda verde clara e, no :hover, fundo branco e sombra leve — combina com o header verde-claro do perfil.

estilo.css
html {
  scroll-behavior: smooth;
}

.perfil-hero__nav {
  display: flex;
  flex-wrap: wrap;
  justify-content: center;
  gap: 0.5rem 0.75rem;
  margin-top: 1.25rem;
}

.perfil-hero__nav a {
  border-radius: 999px;
  padding: 0.4rem 0.95rem;
  border: 1px solid #a7f3d0;
  transition: background 0.2s, box-shadow 0.2s;
}
.perfil-hero__nav a:hover {
  background: #fff;
  box-shadow: 0 2px 8px rgba(4, 120, 87, 0.12);
}

PASSO 1: CSS · MENSAGEM NO RODAPÉ

O parágrafo #msg-visita usa a classe perfil-rodape__extras: mesmo quando o texto ainda não foi escrito pelo JS, a caixa já tem aparência de “avisinho” — fundo verde bem claro, borda suave e cantos arredondados, alinhada ao restante da página.

No mesmo arquivo, .perfil-rodape__extras:empty { display: none } evita mostrar a caixa se o parágrafo estiver vazio (por exemplo sem texto e sem o JS ter rodado).

estilo.css
.perfil-rodape__extras {
  margin: 1.25rem 0 0.75rem;
  padding: 0.85rem 1rem;
  background: #ecfdf5;
  border-radius: 10px;
  border: 1px solid #a7f3d0;
  font-size: 0.95rem;
  color: #374151;
}

.perfil-rodape__extras:empty {
  display: none;
}

PASSO 1: CSS · BOTÃO VOLTAR AO TOPO

O botão fica fixo no canto inferior direito (position: fixed + z-index), em formato de círculo, cor verde escura da paleta e sombra para “levantar” da página. No :hover escurece um pouco e aumenta levemente de tamanho; no :focus-visible aparece contorno verde para quem navega com teclado.

estilo.css
.btn-voltar-topo {
  position: fixed;
  bottom: 1.35rem; right: 1.35rem;
  width: 3rem; height: 3rem;
  border: none; border-radius: 50%;
  background: #047857; color: #fff;
  cursor: pointer;
  box-shadow: 0 4px 14px rgba(4, 120, 87, 0.45);
  z-index: 60;
  display: flex; align-items: center;
  justify-content: center;
  transition: background 0.2s, transform 0.2s;
}

.btn-voltar-topo:hover {
  background: #065f46;
  transform: scale(1.05);
}

.btn-voltar-topo:focus-visible {
  outline: 3px solid #10b981;
  outline-offset: 3px;
}

PASSO 2: ANO NO RODAPÉ

perfil.js
var anoSpan = document.getElementById('rodape-ano');

if (anoSpan) {
  anoSpan.textContent = String(new Date().getFullYear());
}

No HTML o ano começa vazio — o JS preenche ao carregar:

index.html
© <span id="rodape-ano"></span> Helena …

O if (anoSpan) evita erro se o id não existir na página.

PASSO 3: BOTÃO VOLTAR AO TOPO

Mesma lógica do perfil.js: referências, função no scroll, listener passive e clique.

perfil.js (trecho)
var btnTopo = document.getElementById('btn-voltar-topo');
var alvoTopo = document.getElementById('topo');

if (btnTopo && alvoTopo) {
  function atualizaBotaoTopo() {
    var scrollou = window.scrollY > 300;
    btnTopo.hidden = !scrollou;
    btnTopo.setAttribute('aria-hidden', scrollou ? 'false' : 'true');
  }
  atualizaBotaoTopo();
  window.addEventListener('scroll', atualizaBotaoTopo, { passive: true });

  btnTopo.addEventListener('click', function () {
    alvoTopo.scrollIntoView({ behavior: 'smooth', block: 'start' });
  });
}

PASSO 4: sessionStorage

Texto em #msg-visita conforme a primeira carga na aba; try/catch cobre navegação privada ou bloqueio.

perfil.js (trecho)
var msgVisita = document.getElementById('msg-visita');

if (msgVisita) {
  try {
    if (!sessionStorage.getItem('perfilVisitou')) {
      sessionStorage.setItem('perfilVisitou', '1');
      msgVisita.textContent = 'Obrigada por visitar…';
    } else {
      msgVisita.textContent = 'Nesta aba você já abriu…';
    }
  } catch (err) {
    msgVisita.textContent = '';
    msgVisita.style.display = 'none';
  }
}
