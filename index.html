BLOCOS GENÉRICOS: DIV E SPAN

<div>

Elemento de Bloco. Ocupa 100% da largura. Usado para criar caixas e agrupar elementos.

<span>

Elemento em Linha. Ocupa apenas o espaço do texto. Usado para destacar partes de um parágrafo.

SEMÂNTICA: MAIS QUE DIVS

Pare de usar div para tudo! Use tags que descrevam o conteúdo:

  • <header>: Cabeçalho do site.
  • <nav>: Menu de navegação.
  • <main>: Conteúdo principal único.
  • <footer>: Rodapé da página.

ORGANIZAÇÃO: SECTION E ARTICLE

<section>
  <h2>Sobre Mim</h2>
</section>

<article>
  <p>Notícia Independente</p>
</article>

Section: Agrupa conteúdos relacionados por tema.

Article: Conteúdo independente que faz sentido sozinho (Ex: post de blog).

TABELAS PROFISSIONAIS

Use tags organizadoras:

DADOS TABULARES: O QUE É?

Uma tabela HTML representa informações em linhas e colunas: cada <tr> é uma linha; dentro dela, cada <td> ou <th> é uma célula.

Em uma mesma linha, o número de células deve “fechar a conta” com as demais linhas (salvo quando você usa colspan ou rowspan).

Exemplo visual (renderizado):

ProdutoQtdPreço
Caneta10R$ 2,00
Caderno3R$ 15,00

CAPTION, THEAD E TBODY

Abaixo, o código HTML da mesma tabela do slide anterior (Produto / Qtd / Preço). <caption> é o título visível; thead agrupa o cabeçalho; tbody agrupa as linhas de dados.

<table>
  <caption>Itens do pedido</caption>
  <thead>
    <tr>
      <th scope="col">Produto</th><th scope="col">Qtd</th><th scope="col">Preço</th>
    </tr>
  </thead>
  <tbody>
    <tr><td>Caneta</td><td>10</td><td>R$ 2,00</td></tr>
    <tr><td>Caderno</td><td>3</td><td>R$ 15,00</td></tr>
  </tbody>
  <!-- em seguida entra o <tfoot> … -->

No próximo slide: fechamos com tfoot e o </table> correto.

TFOOT — CONCEITO E EXEMPLO VISUAL

<tfoot> agrupa a última linha da tabela — no nosso exemplo, os totais depois de Caneta e Caderno. No HTML ela vem depois do tbody; no navegador o bloco aparece no fim da grade, como rodapé.

Exemplo visual — a linha destacada no fim é o tfoot (totais de quantidade e valor):

Itens do pedido
Produto Qtd Preço
Caneta 10 R$ 2,00
Caderno 3 R$ 15,00
Total 13 R$ 65,00

No próximo slide: o código HTML completo dessa mesma tabela.

TFOOT — CÓDIGO HTML

Mesmo exemplo (Itens do pedido): o bloco <tfoot> no final do arquivo, antes de </table>.

<table>
  <caption>Itens do pedido</caption>
  <thead><tr>
    <th scope="col">Produto</th><th scope="col">Qtd</th><th scope="col">Preço</th>
  </tr></thead>
  <tbody>
    <tr><td>Caneta</td><td>10</td><td>R$ 2,00</td></tr>
    <tr><td>Caderno</td><td>3</td><td>R$ 15,00</td></tr>
  </tbody>
  <tfoot>
    <tr>
      <th scope="row">Total</th><td>13</td><td>R$ 65,00</td>
    </tr>
  </tfoot>
</table>

A linha destacada no slide anterior = este <tfoot>: th com scope="row" em “Total” e duas td (13 un.; R$ 65,00 = 10×R$ 2,00 + 3×R$ 15,00).

POR QUE SEPARAR thead, tbody E tfoot?

  • Acessibilidade: leitores de tela usam essas regiões para anunciar o cabeçalho certo ao ler cada célula — especialmente em tabelas grandes.
  • CSS: você pode estilizar só o corpo (tbody tr:nth-child(even) para zebrar linhas) sem afetar o cabeçalho.
  • Scripts: somar ou filtrar dados costuma olhar só para as linhas dentro de tbody.
  • Impressão: em alguns casos o navegador pode repetir thead (e tratar tfoot) de forma mais previsível ao imprimir páginas longas.
  • Ordem no HTML: mesmo com tfoot, você pode escrever thead → tbody → tfoot; o rodapé continua visualmente no fim da tabela.

<th> VS <td> E O scope

Use <th> para células que são título de linha ou coluna (não só estética: significa “cabeçalho”).

O atributo scope deixa explícito o papel da célula:

  • scope="col" — título da coluna (típico na primeira linha do thead).
  • scope="row" — título da linha (ex.: primeira célula de cada linha no corpo, como “Produto A”).
<tbody>
  <tr>
    <th scope="row">Manhã</th>
    <td>9h–12h</td>
  </tr>
</tbody>

TABELAS: BOAS PRÁTICAS

USE PARA

Preços, horários, comparativos, listas de contatos em grade — sempre que a relação linha × coluna fizer sentido para os dados.

EVITE

Montar o layout geral do site (menu, colunas da página) com tabelas. Hoje usamos CSS Flexbox/Grid para layout; tabela é para dados tabulares.

Combine caption, cabeçalhos th com scope e regiões thead/tbody/tfoot para código claro e mais acessível.

MESCLAGEM: SPAN

<td colspan="2">Ocupa 2 Colunas</td>
<td rowspan="2">Ocupa 2 Linhas</td>

Use colspan para esticar para o lado e rowspan para esticar para baixo.

Cada linha deve somar o mesmo número de colunas lógicas: colspan="2" ocupa dois espaços naquela linha. Com rowspan, a célula “segura” as linhas de baixo — nas próximas <tr> você não coloca outra célula no mesmo lugar, senão a grade quebra.

TUTORIAL: SEU PERFIL

Linha a linha, construindo seu portfólio inicial.

PASSO 1: ARQUIVO

Abra o VS Code, crie index.html e use ! + Enter.

<!DOCTYPE html>
<html lang="pt-br">
  <head>...</head>
  <body>
  </body>
</html>

PASSO 2: ESTRUTURA

Adicione <header> e <main> dentro do body para separar o topo do conteúdo principal.

<body>
  <header>
  </header>
  <main>
  </main>
</body>

PASSO 3: IDENTIDADE

Coloque seu nome principal usando um <h1> dentro do cabeçalho (header).

<header>
  <h1>Seu Nome Aqui</h1>
</header>

PASSO 4: SEÇÃO BIO

Crie uma <section> para agrupar as informações sobre você com um texto inicial.

<main>
  <section>
    <p>Sou estudante de Web.</p>
  </section>
</main>

PASSO 5: MÍDIA

Dentro da mesma <section>, adicione sua foto. Não se esqueça do alt!

<section>
  <img src="foto.jpg"
       alt="Minha foto">
</section>

PASSO 6: SKILLS

Crie uma nova seção ou use a lista <ul> diretamente para enumerar suas habilidades.

<ul>
  <li>HTML5</li>
  <li>CSS3</li>
</ul>

PASSO 7: CONTATOS

Crie um rodapé com <footer> abaixo da <main> e insira links sociais.

<footer>
  <a href="link.com">Meu Link</a>
</footer>
</body>

PASSO 8: TABELA DE CONTATOS

<thead> guarda a primeira linha com os títulos das colunas: use <th scope="col"> em “Rede” e “Link”. <tbody> guarda uma linha por contato: em cada <tr>, a primeira <td> é o nome da rede; a segunda <td> contém o <a href="…"> (URL com https:// ou mailto: para e-mail).

<table>
  <thead>
    <tr>
      <th scope="col">Rede</th>
      <th scope="col">Link</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>GitHub</td>
      <td><a href="https://github.com/seuusuario">Perfil</a></td>
    </tr>
  </tbody>
</table>

Dica: mailto:voce@email.com no href abre o programa de e-mail do visitante.
